02 — Artificial Intelligence
🤖
Groq API — Llama 3.3 70B Versatile
Large Language Model inference · REST API · 30 req/min miễn phí
Chat AI có context học viênAI
System prompt động: inject tên học viên, lớp, tiến độ %, điểm TB, chương đang học vào mỗi request. Giữ 12 tin nhắn gần nhất (sliding window context). Toggle web search qua DuckDuckGo khi cần thông tin mới.
→ File: chat.html · Endpoint: api.groq.com/openai/v1/chat/completions
Chấm điểm vấn đáp — Rubric 4 tiêu chíAI
Prompt engineering yêu cầu AI trả về JSON cấu trúc: {"rubric":{"noi_dung":8,"lap_luan":7,"vi_du":5,"dien_dat":9},"score":7.2,"comment":"..."}. Tính điểm theo trọng số: Nội dung 40% + Lập luận 25% + Ví dụ 20% + Diễn đạt 15%.
→ File: thi-thu.html · max_tokens: 300 · temperature: 0.5
Sinh câu hỏi phụ vấn đápAI
Sau khi học viên trả lời câu chính, Groq phân tích câu trả lời và sinh 2–3 câu hỏi phụ để đào sâu hơn. Trả về JSON array: {"questions":[{"q":"..."},...]}
→ File: thi-thu.html · max_tokens: 400
Phân tích & cấu trúc hóa PDF bài giảngAI
Nhận raw text từ PDF (tối đa 8000 ký tự đầu tiên), phân tích và trả về JSON: chapters[] mỗi chapter có title, segments[], questions[]. AI tự nhận biết cấu trúc chương mục từ văn bản thô.
→ File: giang-bai.html · max_tokens: 2000 · temperature: 0.3
Kiểm tra kiến thức giữa bài giảngAI
Dừng giảng sau mỗi chương → AI hỏi 2 câu kiểm tra → học viên trả lời bằng giọng nói (Speech-to-Text) → AI đánh giá và đọc phản hồi bằng TTS.
→ File: giang-bai.html
03 — Backend & Database
🗄️
Supabase
PostgreSQL as a Service · BaaS · Free tier 500MB
PostgreSQL DatabaseDB
8 bảng chính: profiles, questions, oral_questions, exam_results, quiz_results, oral_results, study_logs, wrong_answers. 1 view tổng hợp: admin_student_summary. JSONB cho rubric_scores linh hoạt.
Row Level Security (RLS)DB
Bảo mật cấp dòng dữ liệu trong PostgreSQL. Mỗi bảng có policy riêng: học viên chỉ đọc/ghi dữ liệu của mình, giảng viên đọc được toàn bộ. Không thể bypass dù gọi API trực tiếp.
Supabase AuthDB
JWT-based authentication. Email/password signup với email verification. Session management tự động refresh token. Trigger tự động tạo profile khi đăng ký.
Supabase JS Client v2Free
SDK JavaScript load từ CDN. Cung cấp: sb.auth.*, sb.from().select/insert/update/delete, realtime subscriptions. Dùng chung qua supabase-config.js trong toàn dự án.
→ Load từ: cdn.jsdelivr.net/npm/@supabase/supabase-js
🔊
Web Speech API
Browser built-in · Không cần key · Chrome/Edge
SpeechSynthesis (TTS)Free
Text-to-Speech built-in trình duyệt. Đọc bài giảng từng đoạn bằng giọng tiếng Việt (vi-VN). Điều chỉnh: tốc độ 0.8×/1×/1.3×, chọn giọng nam/nữ. Dừng/tiếp tục/bỏ qua đoạn.
→ API: window.speechSynthesis · SpeechSynthesisUtterance
SpeechRecognition (STT)Free

→ API: webkitSpeechRecognition · lang: vi-VN

06 — Kiến trúc & Design Patterns
Shared Config Module
supabase-config.js load một lần, expose các helper function toàn cục: requireAuth(), getProfile(), saveExamResult(), logStudyTime()... Tất cả trang HTML đều include file này trước mọi script khác.
→ supabase-config.js
Guard Pattern — Route Protection
Mỗi trang bắt đầu bằng requireAuth() — nếu không có session thì redirect về auth.html. Trang admin.html thêm check role = teacher, nếu là student thì redirect về index.html.
→ Tất cả trang HTML, đặc biệt admin.html
Fallback Content Strategy
Mọi tính năng đều có fallback: Groq fail → dùng câu trả lời cứng built-in. Supabase trống → dùng 40 câu hỏi built-in. TTS không có giọng Việt → dùng giọng mặc định. Hệ thống không bao giờ crash.
→ chat.html, luyen-tap.html, thi-thu.html, giang-bai.html
Batch Insert với Progress Bar
Import hàng trăm câu hỏi không gửi một lần (timeout). Chia thành batch 50 câu, insert tuần tự, cập nhật progress bar sau mỗi batch. Tránh rate limit Supabase.
→ admin.html — import Excel
Dynamic System Prompt
Thay vì system prompt tĩnh, buildSystemPrompt(profile) inject realtime data: tên học viên, lớp, tiến độ, điểm TB, streak, chương đang học → AI trả lời cá nhân hóa hơn.
→ chat.html
Structured JSON Prompting
Mọi Groq call cần dữ liệu có cấu trúc đều yêu cầu AI trả về JSON thuần, không markdown. Sau đó parse và validate trước khi dùng. Fallback nếu JSON parse fail.
→ thi-thu.html, giang-bai.html
07 — Bảo mật
🔐
Row Level Security
PostgreSQL RLS policies trong Supabase. Mỗi user chỉ thấy dữ liệu của mình. Teacher thấy tất cả. Không thể bypass bằng cách gọi REST API trực tiếp — enforce ở database level.
🎫
JWT Authentication
Supabase Auth dùng JWT token. Token tự động refresh. Groq API key lưu trong Supabase profiles (mã hóa) thay vì plain text trong localStorage. Mỗi request đính kèm Bearer token.
👁️
Anon Key vs Service Key
Chỉ dùng anon key trong frontend — key này bị giới hạn bởi RLS. Service key (full access) không bao giờ expose ra client. Mọi quyền phân cấp được enforce qua RLS, không phải client code.
